# dsc_bio — Claude Code Guide

Personal site for David Chelimo, hosted on GitHub Pages at `dchelimo.afyagateway.io`.

---

## Site structure

| URL | Page | Source template |
|---|---|---|
| `/` | Bio (placeholder/landing) | `templates/bio.html` |
| `/cv/` | Full CV / résumé | `templates/cv.html` |
| `/projects/` | Projects (placeholder) | `templates/projects.html` |
| `/blog/` | Blog (placeholder) | `templates/blog.html` |

Each template extends `templates/base.html` (shared `<head>`, top accent bar, nav, footer).

---

## Stack

| Layer | Tool |
|---|---|
| Templating | Jinja2 (Python) — `scripts/build_site.py` |
| Styles | Tailwind CSS v3 (compiled via CLI) |
| Fonts | DM Serif Display · DM Sans · JetBrains Mono (Google Fonts) |
| PDF | WeasyPrint 66.0 — reads `cv/index.html` |
| CI | GitHub Actions — rebuilds site + CSS + PDF on every push to `main` |
| Hosting | GitHub Pages — custom domain via `CNAME` |

## Local dev workflow

```bash
# Install (first time)
npm install
python3 -m venv .venv && .venv/bin/pip install -r requirements.txt

# Edit templates, then build everything:
.venv/bin/python scripts/build_site.py     # render Jinja → HTML
npm run build:css                          # compile Tailwind → dist/styles.css
.venv/bin/python scripts/generate_pdf.py   # regenerate PDF from cv/index.html

# Or watch CSS during dev
npm run watch
```

## PDF print styles

PDF renders from `cv/index.html` only (other pages are not converted). Key rules in `src/input.css` `@media print`:

- **Hidden in PDF:** site bar, site nav, site footer, header links, About section, Selected Work, Languages
- **Shown in PDF:** header summary, all 5 experience entries (earlier experience expanded), Skills, Education, Publications
- **Bullets capped at 3 per experience entry**
- **Layout:** `display: table` two-column (59% left / 41% right)
- **Page:** A4, margins 10mm/13mm
- Gradient name replaced with solid navy

## GitHub Actions (`build.yml`)

Triggers on push to `main` when `templates/**`, `src/**`, `tailwind.config.js`, `scripts/**`, `package.json`, `package-lock.json`, or `requirements.txt` changes.

Steps:
1. Checkout repo
2. Node 20 → `npm ci` → `npm run build:css`
3. Python 3.11 → install WeasyPrint system libs → `pip install -r requirements.txt`
4. `python scripts/build_site.py` (render templates)
5. `python scripts/generate_pdf.py`
6. Commit and push back: `dist/styles.css`, generated HTML files, `David-Chelimo-Resume.pdf`

Requires `permissions: contents: write` on the job.

---

## Adding a new page

1. Add `templates/yourpage.html` extending `base.html`
2. Add link in `templates/base.html` nav block
3. Add tuple to `PAGES` list in `scripts/build_site.py`
4. Run build locally to verify, push — Actions does the rest
