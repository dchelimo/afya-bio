# dsc_bio — Claude Code Guide

Personal site for David Chelimo, hosted on GitHub Pages at `dchelimo.afyagateway.io`.

---

## Site structure

| URL | Page | Source template | In nav? |
|---|---|---|---|
| `/` | Bio + links | `templates/bio.html` | Yes |
| `/cv/` | Full CV / résumé | `templates/cv.html` | No — linked from Bio |
| `/cv/print/` | PDF source (noindex, internal) | `templates/cv_print.html` | No — read by `scripts/generate_pdf.py` |
| `/projects/` | Projects (placeholder) | `templates/projects.html` | Yes |
| `/blog/` | Blog (placeholder) | `templates/blog.html` | Yes |

Nav has three items: **Bio · Projects · Blog**. The CV page exists and is reachable via the "View CV" button on Bio, but is not in the top nav.

Each template extends `templates/base.html` (shared `<head>`, top accent bar, nav, footer). `cv_print.html` is the exception — it's standalone (no nav/footer) because it's the PDF source, not a user-facing page.

## Stack

| Layer | Tool |
|---|---|
| Templating | Jinja2 (Python) — `scripts/build_site.py` |
| CV content | `data/cv.yaml` — single source for both the web CV and the PDF |
| Styles | Tailwind CSS v3 (compiled via CLI) |
| Fonts | DM Serif Display · DM Sans · JetBrains Mono (Google Fonts) |
| PDF | WeasyPrint 66.0 — reads `cv/print/index.html` |
| CI | GitHub Actions — rebuilds site + CSS + PDF on every push to `main` |
| Hosting | GitHub Pages — custom domain via `CNAME` |

## CV content & PDF generation

Both the web CV and the PDF render from `data/cv.yaml`. Two templates pick what to show:

- `templates/cv.html` (web) — extends `base.html`, includes every section.
- `templates/cv_print.html` (PDF) — standalone, includes only items where `pdf` is not `false`.

**Per-item visibility flags** (in `data/cv.yaml`):
- `pdf: false` on a section, entry, or bullet → hidden from PDF
- `web: false` → hidden from web
- Absent → visible in both

The build script (`scripts/build_site.py`) normalizes bullets: a YAML string becomes `{text, pdf: true, web: true}`; an object form `{text, pdf?, web?}` is preserved.

**Currently `pdf: false` items**: `about`, `selected_work`, `languages` sections, plus a few bullets on the Foundation Medicine entries (kept off the PDF to maintain a one-page resume).

**Print CSS in `src/input.css` `@media print`** controls visual layout only — typography, page sizing, two-column table layout, page-break rules. There are no content-hiding rules; visibility is template-driven.

- **Layout:** `display: table` two-column (59% left / 41% right)
- **Page:** A4, margins 10mm/13mm
- Gradient name replaced with solid navy
- Page breaks: `.exp-entry` is unbreakable; sections may split across pages

## GitHub Actions (`build.yml`)

Triggers on push to `main` when `templates/**`, `src/**`, `tailwind.config.js`, `scripts/**`, `package.json`, `package-lock.json`, or `requirements.txt` changes.

Steps:
1. Checkout repo
2. Node 20 → `npm ci` → `npm run build:css`
3. Python 3.11 → install WeasyPrint system libs → `pip install -r requirements.txt`
4. `python scripts/build_site.py` (render templates)
5. `python scripts/generate_pdf.py`
6. Commit and push back: `dist/styles.css`, generated HTML files, `David-Chelimo-Resume.pdf`

Requires `permissions: contents: write` on the job.

## Adding a new page

1. Add `templates/yourpage.html` extending `base.html`
2. Add link in `templates/base.html` nav block
3. Add tuple to `PAGES` list in `scripts/build_site.py`
4. Run build locally to verify, push — Actions does the rest

## Editing CV content

For typos, date changes, new bullets, or hiding an item from one format: edit `data/cv.yaml`. Don't touch `templates/cv.html` or `templates/cv_print.html` unless the change is structural (a new section type, new field, new layout). To hide a bullet from the PDF only:

```yaml
bullets:
  - Plain string bullet — shown in both web and PDF
  - text: This bullet is web-only
    pdf: false
```
