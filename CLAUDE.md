# dsc_bio — Claude Code Guide

Personal bio/resume site for David Chelimo, hosted on GitHub Pages at `dchelimo.afyagateway.io`.

## Migration plan: Tailwind CSS + WeasyPrint PDF

### Goal
Replace the inline `<style>` block in `index.html` with Tailwind CSS utility classes, and replace the client-side `html2pdf.js` button with a statically generated PDF produced by WeasyPrint via GitHub Actions.

---

### Step 1 — Set up Tailwind CLI

1. Initialise a `package.json`:
   ```bash
   npm init -y
   ```
2. Install Tailwind CSS and its CLI:
   ```bash
   npm install -D tailwindcss
   npx tailwindcss init
   ```
3. Create `src/input.css`:
   ```css
   @tailwind base;
   @tailwind components;
   @tailwind utilities;
   ```
4. Configure `tailwind.config.js` to scan `index.html` for class names:
   ```js
   content: ["./index.html"],
   ```
5. Add a build script to `package.json`:
   ```json
   "scripts": {
     "build:css": "tailwindcss -i ./src/input.css -o ./dist/styles.css --minify",
     "watch": "tailwindcss -i ./src/input.css -o ./dist/styles.css --watch"
   }
   ```
6. Link `dist/styles.css` from `index.html` and remove the inline `<style>` block.

---

### Step 2 — Migrate styles to Tailwind utility classes

- Convert every CSS rule in the current `<style>` block to Tailwind utility classes on the HTML elements.
- Key custom values (gradient colours, card radii) go in `tailwind.config.js` under `theme.extend`.
- Keep a `@layer components` block in `src/input.css` for any patterns that repeat enough to warrant a class (e.g. `.card`, `.tag`, `.pill-tag`).

---

### Step 3 — Set up WeasyPrint locally

1. Install WeasyPrint:
   ```bash
   pip install weasyprint
   ```
2. Create `scripts/generate_pdf.py`:
   - Reads `index.html` and `dist/styles.css`
   - Strips the PDF button and any screen-only elements
   - Outputs `David-Chelimo-Resume.pdf` to the repo root
3. Test locally:
   ```bash
   python scripts/generate_pdf.py
   ```
4. Verify page breaks, margins, and font rendering in the output PDF.

---

### Step 4 — GitHub Actions workflow

Create `.github/workflows/build.yml` that runs on every push to `main`:

1. Check out the repo.
2. Set up Node.js and run `npm run build:css` to compile Tailwind.
3. Set up Python and install WeasyPrint (`pip install weasyprint`).
4. Run `python scripts/generate_pdf.py`.
5. Commit `dist/styles.css` and `David-Chelimo-Resume.pdf` back to `main` if they changed.

---

### Step 5 — Update PDF button

- Change the "PDF Resume" link from `onclick="downloadPDF()"` back to `href="David-Chelimo-Resume.pdf"`.
- Remove the `html2pdf.js` script tag and `downloadPDF()` function.

---

### Step 6 — Verify and clean up

- Confirm the live site loads correctly from `dist/styles.css`.
- Confirm the PDF button downloads the Actions-generated file.
- Add `node_modules/` and `dist/` to `.gitignore` (Actions generates `dist/styles.css` at build time; don't commit it from local).
- Remove `html2pdf.js` CDN reference.

---

## Local dev workflow (after migration)

```bash
npm run watch        # recompile CSS on file save
python scripts/generate_pdf.py  # regenerate PDF locally
git add . && git commit && git push  # Actions will rebuild on push
```
